# 一、MVP 数据库设计目标

你的第一版系统要跑通这条链：

**项目 → 资料 → 文档切片 → 知识点 → 计划 → 任务 → 学习会话 → 测验 → 复习**

所以数据库至少要覆盖这些核心对象：

- 用户
    
- 学习项目
    
- 资料
    
- 文档切片
    
- 知识单元
    
- 学习计划
    
- 学习任务
    
- 学习会话
    
- 笔记
    
- 测验
    
- 测验答案
    
- 复习记录

# 二、表关系总览

```text
User
 └── Project
      ├── Material
      │    ├── MaterialChunk
      │    └── KnowledgeUnit
      ├── StudyPlan
      │    └── Task
      │         ├── LearningSession
      │         ├── Note
      │         ├── Quiz
      │         │    └── QuizAnswer
      │         └── ReviewRecord
      └── ProjectKnowledgeUnit (可选，后续扩展)
```

实际 MVP 中可以先简化为：

- `Project` 下面挂 `Material`
    
- `Material` 下面产出 `MaterialChunk` 和 `KnowledgeUnit`
    
- `StudyPlan` 下面挂 `Task`
    
- `Task` 下面挂学习、测验、复习等记录

# 三、Prisma schema 初稿

下面是一版比较适合直接开工的 `schema.prisma` 初稿。

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

enum MaterialType {
  PDF
  MARKDOWN
  TEXT
}

enum MaterialStatus {
  UPLOADING
  PARSING
  READY
  FAILED
}

enum PlanStatus {
  DRAFT
  ACTIVE
  COMPLETED
  ARCHIVED
}

enum TaskType {
  LEARN
  REVIEW
  REINFORCE
  QUIZ
}

enum TaskStatus {
  TODO
  IN_PROGRESS
  DONE
  SKIPPED
  OVERDUE
}

enum SessionStatus {
  STARTED
  COMPLETED
  ABANDONED
}

enum QuizStatus {
  GENERATED
  SUBMITTED
  GRADED
}

enum ReviewResult {
  POOR
  FAIR
  GOOD
  EXCELLENT
}

model User {
  id          String    @id @default(cuid())
  email       String    @unique
  name        String?
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  projects    Project[]
}

model Project {
  id               String       @id @default(cuid())
  userId           String
  name             String
  description      String?
  goal             String?
  examDate         DateTime?
  dailyTimeBudget  Int?         // 单位：分钟
  mode             String?      // 例如: steady / sprint / light
  createdAt        DateTime     @default(now())
  updatedAt        DateTime     @updatedAt

  user             User         @relation(fields: [userId], references: [id], onDelete: Cascade)
  materials        Material[]
  studyPlans       StudyPlan[]

  @@index([userId])
}

model Material {
  id               String          @id @default(cuid())
  projectId        String
  title            String
  type             MaterialType
  sourcePath       String?         // 文件路径或对象存储地址
  rawText          String?
  status           MaterialStatus  @default(UPLOADING)
  parseError       String?
  metadata         Json?
  createdAt        DateTime        @default(now())
  updatedAt        DateTime        @updatedAt

  project          Project         @relation(fields: [projectId], references: [id], onDelete: Cascade)
  chunks           MaterialChunk[]
  knowledgeUnits   KnowledgeUnit[]

  @@index([projectId])
  @@index([status])
}

model MaterialChunk {
  id               String    @id @default(cuid())
  materialId       String
  chunkIndex       Int
  title            String?
  content          String
  summary          String?
  sourcePage       Int?
  sourceSection    String?
  metadata         Json?
  embedding        Unsupported("vector")?
  createdAt        DateTime  @default(now())

  material         Material  @relation(fields: [materialId], references: [id], onDelete: Cascade)

  @@unique([materialId, chunkIndex])
  @@index([materialId])
}

model KnowledgeUnit {
  id                String     @id @default(cuid())
  materialId        String
  title             String
  summary           String?
  topic             String?
  difficulty        Int?       // 1~5
  importance        Int?       // 1~5
  estimatedMinutes  Int?
  prerequisiteIds   String[]   // MVP 先简单存字符串数组
  sourceRefs        Json?      // 记录关联 chunk/page/section
  createdAt         DateTime   @default(now())
  updatedAt         DateTime   @updatedAt

  material          Material   @relation(fields: [materialId], references: [id], onDelete: Cascade)
  taskLinks         TaskKnowledgeUnit[]

  @@index([materialId])
  @@index([topic])
}

model StudyPlan {
  id               String      @id @default(cuid())
  projectId        String
  title            String
  startDate        DateTime
  endDate          DateTime
  totalDays        Int
  status           PlanStatus  @default(DRAFT)
  config           Json?       // 记录生成计划时使用的参数
  createdAt        DateTime    @default(now())
  updatedAt        DateTime    @updatedAt

  project          Project     @relation(fields: [projectId], references: [id], onDelete: Cascade)
  tasks            Task[]

  @@index([projectId])
  @@index([status])
}

model Task {
  id                String       @id @default(cuid())
  studyPlanId       String
  title             String
  description       String?
  taskType          TaskType
  status            TaskStatus   @default(TODO)
  scheduledDate     DateTime
  estimatedMinutes  Int?
  difficulty        Int?
  priority          Int?         // 1~5
  masteryScore      Float?
  nextReviewDate    DateTime?
  createdAt         DateTime     @default(now())
  updatedAt         DateTime     @updatedAt
  completedAt       DateTime?

  studyPlan         StudyPlan    @relation(fields: [studyPlanId], references: [id], onDelete: Cascade)
  knowledgeLinks    TaskKnowledgeUnit[]
  sessions          LearningSession[]
  notes             Note[]
  quizzes           Quiz[]
  reviewRecords     ReviewRecord[]

  @@index([studyPlanId])
  @@index([scheduledDate])
  @@index([status])
  @@index([taskType])
  @@index([nextReviewDate])
}

model TaskKnowledgeUnit {
  id               String         @id @default(cuid())
  taskId           String
  knowledgeUnitId  String

  task             Task           @relation(fields: [taskId], references: [id], onDelete: Cascade)
  knowledgeUnit    KnowledgeUnit  @relation(fields: [knowledgeUnitId], references: [id], onDelete: Cascade)

  @@unique([taskId, knowledgeUnitId])
  @@index([knowledgeUnitId])
}

model LearningSession {
  id               String         @id @default(cuid())
  taskId           String
  startedAt        DateTime       @default(now())
  endedAt          DateTime?
  durationSeconds  Int?
  questionCount    Int            @default(0)
  status           SessionStatus  @default(STARTED)
  createdAt        DateTime       @default(now())

  task             Task           @relation(fields: [taskId], references: [id], onDelete: Cascade)

  @@index([taskId])
  @@index([status])
}

model Note {
  id               String    @id @default(cuid())
  taskId           String
  content          String
  isImportant      Boolean   @default(false)
  isConfusing      Boolean   @default(false)
  createdAt        DateTime  @default(now())
  updatedAt        DateTime  @updatedAt

  task             Task      @relation(fields: [taskId], references: [id], onDelete: Cascade)

  @@index([taskId])
}

model Quiz {
  id               String      @id @default(cuid())
  taskId           String
  title            String?
  status           QuizStatus  @default(GENERATED)
  score            Float?
  feedback         Json?
  generatedAt      DateTime    @default(now())
  submittedAt      DateTime?
  gradedAt         DateTime?

  task             Task        @relation(fields: [taskId], references: [id], onDelete: Cascade)
  answers          QuizAnswer[]

  @@index([taskId])
  @@index([status])
}

model QuizAnswer {
  id               String    @id @default(cuid())
  quizId           String
  questionIndex    Int
  questionType     String    // recall / understanding / application / transfer
  questionText     String
  referenceAnswer  String?
  userAnswer       String?
  isCorrect        Boolean?
  score            Float?
  analysis         String?
  errorType        String?   // memory / confusion / understanding / application
  createdAt        DateTime  @default(now())

  quiz             Quiz      @relation(fields: [quizId], references: [id], onDelete: Cascade)

  @@unique([quizId, questionIndex])
  @@index([quizId])
}

model ReviewRecord {
  id               String        @id @default(cuid())
  taskId           String
  reviewDate       DateTime      @default(now())
  result           ReviewResult
  score            Float?
  intervalDays     Int?
  nextReviewDate   DateTime?
  notes            String?
  createdAt        DateTime      @default(now())

  task             Task          @relation(fields: [taskId], references: [id], onDelete: Cascade)

  @@index([taskId])
  @@index([reviewDate])
  @@index([nextReviewDate])
}
```

# 四、这版 schema 的设计说明

## 1. 为什么 `KnowledgeUnit` 不直接挂在 `Project`

因为知识点最初来自某份资料，所以 MVP 阶段挂在 `Material` 上更自然。

后面如果你要做“多资料融合成同一知识图谱”，再加一个：

- `ProjectKnowledgeUnit`  
    或者
    
- `KnowledgeGraphNode`
    

第一版不急。

---

## 2. 为什么 `Task` 和 `KnowledgeUnit` 是多对多

因为一个任务可能包含：

- 1 个知识点
    
- 或 2~3 个强相关知识点
    

所以不能简单让 `Task` 只挂一个 `knowledgeUnitId`。

这就是 `TaskKnowledgeUnit` 的意义。

---

## 3. 为什么 `prerequisiteIds` 先用 `String[]`

因为 MVP 阶段你最需要的是先跑通，而不是把依赖图建得非常复杂。

后面如果依赖关系要精细化，可以改成单独关系表：

- `KnowledgeDependency`
    
    - `fromKnowledgeUnitId`
        
    - `toKnowledgeUnitId`
        

第一版先用数组最省事。

---

## 4. 为什么 `MaterialChunk.embedding` 用 `Unsupported("vector")`

因为 Prisma 对 pgvector 的原生支持不如普通字段那样直接，实际项目里通常会：

- 在 Prisma 中先写 `Unsupported("vector")`
    
- 用原生 SQL 执行向量检索
    
- 或借助 Prisma raw query
    

这很常见，适合 MVP。

---

## 5. 为什么 `QuizAnswer` 把题目和答案分开存

因为你后面会需要：

- 单题评分
    
- 单题错因分析
    
- 错题回流
    
- 按题型统计
    

所以不能只把整个 quiz 作为一坨 JSON。

---

# 五、推荐补充的数据库约束和习惯

## 1. 所有核心表都加 `createdAt` / `updatedAt`

你后面调试、统计、做时间线都很有用。

## 2. 核心查询字段都建索引

特别是：

- `projectId`
    
- `studyPlanId`
    
- `taskId`
    
- `scheduledDate`
    
- `status`
    
- `nextReviewDate`
    

## 3. 不要太早做太多 JSON

JSON 适合存：

- 配置
    
- source refs
    
- 模型反馈原始数据
    

但不适合把核心业务字段全塞进去。

# 七、第一版最常用的查询场景

你后面写接口时，最常用的是这些：

## 1. 获取某项目下的资料列表

查：

- `Material`
    

## 2. 获取某资料的解析结构

查：

- `Material`
    
- `MaterialChunk`
    
- `KnowledgeUnit`
    

## 3. 获取某天的任务列表

查：

- `Task`
    
- 关联 `StudyPlan`
    

## 4. 获取任务详情

查：

- `Task`
    
- `TaskKnowledgeUnit`
    
- `KnowledgeUnit`
    
- 相关 `MaterialChunk`
    

## 5. 获取今日待复习任务

查：

- `Task.nextReviewDate <= today`
    
- `Task.status = DONE`
    

## 6. 查看任务测验结果

查：

- `Quiz`
    
- `QuizAnswer`

# 八、后续扩展建议

这版 schema 够你做 MVP，但后面你大概率会扩展这几块。

## 1. 聊天记录表

如果你要保存任务内问答历史，可以加：

- `ChatMessage`
    
    - `taskId`
        
    - `role`
        
    - `content`
        
    - `citations`
        
    - `createdAt`
        

## 2. 知识依赖关系表

如果后面依赖图复杂，增加：

- `KnowledgeDependency`
    

## 3. 错题本表

如果后面你想专门做错题池，可以增加：

- `WrongQuestion`
    
    - `quizAnswerId`
        
    - `taskId`
        
    - `knowledgeUnitId`
        
    - `resolved`
        

## 4. 文件上传元数据表

如果后面支持对象存储、版本管理、hash 去重，可以加强 `Material` 或拆 `FileAsset`

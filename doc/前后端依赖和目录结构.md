
我按你已经定下来的方案来写：

* 前端：React + TypeScript + Vite + React Router + Zustand + TanStack Query + shadcn/ui + Tailwind CSS
* 后端：NestJS + TypeScript + Prisma + PostgreSQL

Vite 官方支持直接创建 React/TypeScript 模板；Tailwind 提供了 Vite 安装方式；React Router、TanStack Query、Zod、Nest CLI、Nest Config、Prisma 也都有对应安装文档。([vitejs][1])

---

# 一、前端依赖

## 1）初始化项目

```bash
npm create vite@latest frontend -- --template react-ts
cd frontend
npm install
```

Vite 文档列出了 `react-ts` 模板，并给出了 `npm create vite@latest` 的初始化方式。([vitejs][1])

## 2）核心依赖

```bash
npm install react-router @tanstack/react-query zustand react-hook-form zod
```

React Router 安装文档给出的包名是 `react-router`；TanStack Query 安装包是 `@tanstack/react-query`；Zod 官方将其定义为 TypeScript-first 的校验库；React Hook Form 和 Zustand 的官方文档也给出了直接安装方式。([reactrouter.com][2])

## 3）UI 与样式依赖

```bash
npm install tailwindcss @tailwindcss/vite
npm install shadcn class-variance-authority clsx tailwind-merge lucide-react tw-animate-css
```

Tailwind 官方提供了 Vite 安装方式；shadcn/ui 的手动安装文档列出了 `shadcn`、`class-variance-authority`、`clsx`、`tailwind-merge`、`lucide-react`、`tw-animate-css` 这组依赖。([Tailwind CSS][3])

## 4）建议补充的开发依赖

```bash
npm install -D @types/node
```

如果你后面会在 Vite 配置、路径别名或 Node 环境变量里用到 Node 类型，这个通常需要装。

## 5）初始化 shadcn/ui

```bash
npx shadcn@latest init -t vite
```

shadcn/ui 的 Vite 安装文档提供了 `init -t vite` 的初始化方式；CLI 文档说明 `init` 会初始化配置、安装依赖并生成 `components.json`。([ui.shadcn.com][4])

# 三、后端依赖

## 1）初始化 NestJS

```bash
npm i -g @nestjs/cli
nest new backend --strict
cd backend
```

Nest 官方文档给出了 `npm i -g @nestjs/cli` 和 `nest new` 的初始化方式，并提到可以用 `--strict` 开启更严格的 TypeScript 配置。([docs.nestjs.com][5])

## 2）核心业务依赖

```bash
npm install @nestjs/config @nestjs/swagger swagger-ui-express
npm install @prisma/client
npm install class-validator class-transformer
npm install pg multer dotenv
```

Nest 的配置文档展示了 `@nestjs/config` 的使用；Nest 的 OpenAPI 文档对应 `@nestjs/swagger` 和 `swagger-ui-express`；Prisma 官方文档要求安装 `@prisma/client` 和 Prisma CLI；PostgreSQL 在 Node 里通常用 `pg` 驱动，文件上传通常会用 `multer`。([docs.nestjs.com][6])

## 3）开发依赖

```bash
npm install -D prisma @types/node @types/multer
```

Prisma Quickstart 明确需要安装 Prisma CLI；Node 类型在 TS 项目里通常都会加。([Prisma][7])

## 4）初始化 Prisma

```bash
npx prisma init
```

Prisma 官方 Quickstart 使用 Prisma CLI 初始化项目。([Prisma][7])

## 5）后端可选依赖

如果你很快就要接 LLM 和文档解析，建议加上：

```bash
npm install axios zod
npm install pdf-parse
```

* `axios`：调模型 API 顺手
* `zod`：用于 AI 输出结构校验也很好用
* `pdf-parse`：先做 PDF 文本提取的 MVP

# 七、最实用的建议

先不要把所有模块一次性建完。
你可以先只建这几个业务模块：

* `project`
* `material`
* `parser`
* `task`
* `chat`
* `quiz`
* `review`

因为这几个已经足够跑通 MVP 主链路。

下一步最合适的是，我直接帮你出一版 **前端 package.json 建议依赖列表 + 后端 package.json 建议依赖列表**，你可以直接复制进去。

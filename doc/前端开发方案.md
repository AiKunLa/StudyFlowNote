## 三、技术架构

### 3.1 技术栈

```
前端框架: React 19 + TypeScript 5.9
构建工具: Vite 7
路由: React Router 7
状态管理:
  - TanStack Query (服务端状态)
  - Zustand (客户端状态)
表单: React Hook Form + Zod
UI组件: shadcn/ui + Radix UI
样式: Tailwind CSS 3.4
动画: Framer Motion (可选)
```

## 五、开发原则

### 5.1 组件开发原则

1. **小而专注**：每个组件不超过 200 行
2. **组合优于继承**：使用 composition 模式
3. **受控组件**：表单使用 react-hook-form
4. **错误处理**：每个异步操作都有 loading/error 状态

### 5.2 状态管理原则

1. **服务端状态**：使用 TanStack Query 管理
2. **客户端状态**：使用 Zustand 管理
3. **URL 状态**：分页、筛选等使用 URL 参数

### 5.3 API 对接原则

1. **基于 OpenAPI**：严格遵循 doc/api/openapi.yaml
2. **统一错误处理**：所有请求有统一的 error handling
3. **类型安全**：请求/响应都有类型定义

### 5.4 代码质量

1. **提交前检查**：
   - [ ] TypeScript 类型正确
   - [ ] ESLint 无错误
   - [ ] 组件有 PropTypes/TypeScript 类型
2. **测试覆盖**：
   - 工具函数 100% 覆盖
   - 关键业务逻辑 80%+ 覆盖
